# SubCellAE — Project Context (2026-05-14)

## What this project does

SubCellAE trains autoencoders on fluorescence microscopy patches (32×32 px, paxillin channel) to learn latent representations of focal adhesion (FA) structures. The latent space is then used for downstream FA-type classification (Nascent Adhesion / focal complex / focal adhesion / fibrillar adhesion / No adhesion) and position classification (Cell Protruding Edge / Cell Periphery / Lamella / Cell Body).

**Biological context:** Focal adhesions are cell-substrate attachment structures. The goal is to characterise FA morphology differences between control and Y-compound-treated (ycomp) conditions using unsupervised/semi-supervised learned representations.

## Model variants (model_type in config)

| model_type | Description |
|------------|-------------|
| `ae` | Standard convolutional AE, reconstruction only |
| `vae` | Variational AE with KL loss |
| `semisup` | Semi-supervised AE with FA-type + Position classification heads |
| `contrastive` | Contrastive AE with NT-Xent loss (rotation-invariant embeddings) |

The `semisup` model with dual heads (`lambda_cls_2 > 0`) is the primary model used in experiments — it jointly optimises reconstruction + FA-type classification + position classification.

## Training strategies (the sweep)

Four strategies were historically tested. All use `model_type: semisup`, `latent_dim=8`, `lr=0.001`, `batch_size=128`. Key differences:

| Strategy | Epochs | weight_decay | warmup_epochs | lr_scheduler | Best ckpt |
|----------|--------|-------------|---------------|--------------|-----------|
| `0322` | 200 | 0 | 0 | none | final epoch |
| `0324` | 500 | 1e-4 | 0 | none | after ep 200 |
| `mar30` | 500 | 1e-4 | 200 | none | after ep 200 |
| `final` | 500 | 1e-4 | 200 | cosine → 1e-5 | after ep 200 |

**Key finding:** `0322` overfits severely (train→0, val diverges) but gives the **best downstream FA-type classification accuracy**. The later strategies are more regularised but classify less well. This is the core research tension.

`warmup_epochs` = recon-only phase before the classification heads are activated. After warmup the LR is reset to the original value and the scheduler restarted.

Config files: `config/training_strategies/strategy_{0322,0324,mar30,final}.yaml`
These use `root_folder` variable for path substitution — update this for a new machine.

## AEConfig key parameters (subcellae/pipeline/ae_pipeline.py)

```python
# Training
epochs, lr, batch_size, val_split, loss_norm_flag, group_split
weight_decay        # Adam L2 — now applies to both ae and semisup model types
warmup_epochs       # semisup: recon-only phase before cls heads activate
lr_scheduler        # "none" | "plateau" | "cosine"
lr_min              # floor for cosine/plateau scheduler
early_stopping_patience  # 0 = disabled
min_epochs_for_best      # best-ckpt tracking starts at this epoch

# Model (semisup)
num_classes, lambda_recon, lambda_cls
num_classes_2, lambda_cls_2   # second head (Position); 0 = disabled
```

## Contrastive AE notes (task for exp/contrastive_projector)

The `ContrastiveAE` has a `self.projector` head used only during NT-Xent training. After training, downstream analysis (latent CSV, classification, UMAP) uses raw `z` from the encoder — the projector is discarded (SimCLR-style).

**Identified issue / next task:** The projector projection space `proj(z)` might cluster FA types better than raw `z`. Worth extracting `proj(z)` into the latent CSV and testing. Also: fundamental tension between reconstruction quality (low MSE = every patch detail encoded in z) and latent clustering quality (invariant z = same recon for all rotations = high MSE). Two directions discussed:
1. Split z into FA-type subspace + nuisance subspace (option 2, most tractable)
2. Change reconstruction target to canonical/rotationally-averaged patch
